# 📝 解决开了代理 GitHub 提交还是很慢的问题

**场景：** 电脑明明有代理，网页访问 GitHub 很快，但在终端执行 `git clone` 或 `push` 速度依然只有几 kb/s。 **原因：** Git 终端默认不走系统代理，需要单独配置一下。

### 1. 设置代理（当前仓库生效）

进入你的项目文件夹，执行下面两行（端口号 `7890` 是 Clash 等常见软件的默认端口，如果不一致请自行修改）：

```bash
git config http.proxy socks5://127.0.0.1:7890
git config https.proxy socks5://127.0.0.1:7890
```

### 2. 查看当前代理

看看有没有配置成功：

```bash
git config http.proxy
git config https.proxy
```

### 3. 取消代理

如果不需要了，或者关了梯子报错，用这个命令取消：

```bash
git config --unset http.proxy
git config --unset https.proxy
```

---

### 💡 懒人专用：全局设置（--global）

如果你不想每个项目都敲一遍命令，可以加上 `--global` 参数，这样电脑里所有 Git 仓库都会生效：

```bash
git config --global http.proxy socks5://127.0.0.1:7890
git config --global https.proxy socks5://127.0.0.1:7890
```

**⚠️ 特别注意：** 如果开启了全局代理，当你需要**提交代码到公司内网（GitLab）**或者 **Gitee** 时，Git 也会尝试走代理，大概率会连接失败。这时候记得必须执行取消命令（加上 `--global`）： `git config --global --unset http.proxy`
